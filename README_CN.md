# WebSSH 浏览器 SSH 客户端

一个完全运行在浏览器中的纯 JavaScript SSH 客户端，无需任何服务端 SSH 代理即可建立 SSH 连接。

## 项目介绍

WebSSH 是一个创新的 SSH 客户端实现，它将完整的 SSH 协议栈移植到了浏览器环境中。通过 Web Crypto API 和 WebSocket 技术，实现了与远程服务器的安全通信。

### 主要特性

- **纯浏览器实现**：完整的 SSH 协议在 JavaScript 中实现
- **多种认证方式**：
  - 密码认证
  - 公钥认证（支持 RSA、ECDSA、Ed25519）
- **交互式终端**：基于 xterm.js 的完整终端模拟
- **安全加密**：AES-CTR 加密 + HMAC-SHA256 完整性校验
- **密钥交换**：Diffie-Hellman Group 14 SHA-256

### 支持的算法

| 类型 | 算法 |
|------|------|
| 密钥交换 | diffie-hellman-group14-sha256 |
| 加密 | aes256-ctr |
| MAC | hmac-sha2-256 |
| 公钥 | ssh-rsa, ecdsa-sha2-*, ssh-ed25519 |

### SSH 协议流程

```
客户端                                    服务器
  │                                         │
  │─────── SSH 版本标识 ------------------>│
  │<------ SSH 版本标识 -------------------│
  │                                         │
  │─────── KEXINIT (密钥交换初始化) ------->│
  │<------ KEXINIT ------------------------│
  │                                         │
  │─────── KEXDH_INIT (DH公钥) ------------>│
  │<------ KEXDH_REPLY (DH响应) -----------│
  │         (包含服务器公钥、签名)           │
  │                                         │
  │        [双方计算共享密钥 K]              │
  │        [派生加密密钥和 MAC 密钥]         │
  │                                         │
  │─────── NEWKEYS ------------------------>│
  │<------ NEWKEYS -------------------------│
  │                                         │
  │======== 加密传输层开始 =================│
  │                                         │
  │─────── SERVICE_REQUEST (ssh-userauth) ->│
  │<------ SERVICE_ACCEPT ------------------│
  │                                         │
  │─────── USERAUTH_REQUEST --------------->│
  │         (用户名 + 认证方式)              │
  │<------ USERAUTH_SUCCESS ---------------│
  │         (或 USERAUTH_FAILURE)           │
  │                                         │
  │─────── CHANNEL_OPEN (session) --------->│
  │<------ CHANNEL_OPEN_CONFIRMATION ------│
  │                                         │
  │─────── PTY-REQUEST -------------------->│
  │─────── SHELL-REQUEST ------------------>│
  │<------ CHANNEL_SUCCESS ----------------│
  │                                         │
  │======== 交互式 Shell 会话 ==============│
  │                                         │
  │─────── CHANNEL_DATA (用户输入) -------->│
  │<------ CHANNEL_DATA (服务器输出) -------│
  │         ... 循环 ...                    │
```

### 密钥派生过程

从共享密钥 K 和交换哈希 H 派生出各个密钥：

```
IV_C2S  = SHA256(K || H || "A" || session_id)  // 客户端到服务器 IV
IV_S2C  = SHA256(K || H || "B" || session_id)  // 服务器到客户端 IV
Enc_C2S = SHA256(K || H || "C" || session_id)  // 客户端到服务器加密密钥
Enc_S2C = SHA256(K || H || "D" || session_id)  // 服务器到客户端加密密钥
MAC_C2S = SHA256(K || H || "E" || session_id)  // 客户端到服务器 MAC 密钥
MAC_S2C = SHA256(K || H || "F" || session_id)  // 服务器到客户端 MAC 密钥
```

### 数据包结构

**未加密数据包：**
```
┌────────────┬────────────┬─────────────┬─────────────┐
│  packet    │  padding   │             │   random    │
│  length    │  length    │   payload   │   padding   │
│  (4 bytes) │  (1 byte)  │   (n bytes) │   (m bytes) │
└────────────┴────────────┴─────────────┴─────────────┘
```

**加密数据包：**
```
┌─────────────────────────────────────┬─────────────┐
│          encrypted packet           │     MAC     │
│   (length + padding + payload + pad)│  (32 bytes) │
└─────────────────────────────────────┴─────────────┘
```

### 公钥认证流程

```
1. 客户端发送查询请求
   ┌────────────────────────────────────────┐
   │ USERAUTH_REQUEST                       │
   │ - 用户名                               │
   │ - 认证方式: publickey                  │
   │ - want_reply: false                    │
   │ - 公钥算法                             │
   │ - 公钥数据                             │
   └────────────────────────────────────────┘
                      │
                      ▼
2. 服务器验证公钥是否授权
   ┌────────────────────────────────────────┐
   │ USERAUTH_PK_OK (如果公钥被接受)        │
   │ 或                                     │
   │ USERAUTH_FAILURE (如果公钥未被接受)    │
   └────────────────────────────────────────┘
                      │
                      ▼
3. 客户端签名并发送
   ┌────────────────────────────────────────┐
   │ USERAUTH_REQUEST                       │
   │ - 用户名                               │
   │ - 认证方式: publickey                  │
   │ - want_reply: true                     │
   │ - 公钥算法                             │
   │ - 公钥数据                             │
   │ - 签名:                                │
   │   算法名 + sign(session_id || 请求数据)│
   └────────────────────────────────────────┘
                      │
                      ▼
4. 服务器验证签名
   ┌────────────────────────────────────────┐
   │ USERAUTH_SUCCESS (认证成功)            │
   │ 或                                     │
   │ USERAUTH_FAILURE (认证失败)            │
   └────────────────────────────────────────┘
```

## 使用说明

### 快速开始

1. **启动本地 HTTP 服务器**
   ```bash
   cd /home/level/webssh
   python3 -m http.server 8080
   ```

2. **打开浏览器访问**
   ```
   http://localhost:8080
   ```

3. **填写连接信息**
   - 主机：SSH 服务器 IP 地址
   - 端口：SSH 端口（默认 22）
   - 用户名：SSH 用户名

4. **选择认证方式**
   - 密码认证：输入密码
   - 密钥认证：选择私钥文件

5. **点击「连接」按钮**

### 密钥认证配置

#### 生成 SSH 密钥

```bash
# 生成 Ed25519 密钥（推荐）
ssh-keygen -t ed25519 -f ~/.ssh/id_ed25519

# 生成 RSA 密钥
ssh-keygen -t rsa -b 4096 -f ~/.ssh/id_rsa

# 生成 ECDSA 密钥
ssh-keygen -t ecdsa -b 256 -f ~/.ssh/id_ecdsa
```

#### 上传公钥到服务器

```bash
# 方法一：使用 ssh-copy-id
ssh-copy-id -i ~/.ssh/id_ed25519.pub user@server

# 方法二：手动添加
cat ~/.ssh/id_ed25519.pub | ssh user@server 'cat >> ~/.ssh/authorized_keys'
```

#### 使用私钥登录

1. 在 WebSSH 界面选择「密钥」认证方式
2. 点击「选择私钥」按钮
3. 选择私钥文件（如 `~/.ssh/id_ed25519`）
4. 点击「连接」

### 支持的私钥格式

支持 OpenSSH 私钥格式：

```
-----BEGIN OPENSSH PRIVATE KEY-----
b3BlbnNzaC1rZXktdjEAAAAABG5vbmUAAAAEbm9uZQAAAAAAAAABAAAAMwAAAAtzc2gtZW
...
-----END OPENSSH PRIVATE KEY-----
```

**注意**：暂不支持加密的私钥文件，请使用未加密的私钥。

### 快捷键

终端支持标准快捷键：
- `Ctrl+C`：中断当前命令

### 故障排除

#### 连接失败

1. 检查网络连接和防火墙设置
2. 确认 SSH 服务器正在运行
3. 检查 WebSocket 代理是否正常工作

#### 认证失败

1. 密码认证：确认用户名和密码正确
2. 密钥认证：
   - 确认公钥已添加到服务器的 `~/.ssh/authorized_keys`
   - 确认私钥文件格式正确
   - 确认私钥未加密（暂不支持加密私钥）

#### 终端显示异常

1. 刷新页面重试
2. 检查浏览器控制台是否有错误信息
3. 确认使用的是现代浏览器（Chrome 90+, Firefox 90+, Safari 15+）

## 技术栈

- **前端**：原生 JavaScript + HTML5 + CSS3
- **终端**：xterm.js 5.3.0
- **加密**：
  - Web Crypto API（AES, RSA, ECDSA）
  - TweetNaCl.js（Ed25519）
- **通信**：WebSocket

## 安全说明

- 私钥仅在浏览器内存中处理，不会上传到服务器
- 不存储任何凭证信息
- 使用强加密算法（AES-256-CTR）
- HMAC-SHA256 保证消息完整性
- 页面关闭后所有密钥自动清除

## 已知限制

- 仅支持单个通道（一个 shell 会话）
- 不支持端口转发
- 不支持 X11 转发
- 不支持 agent 转发
- 不支持加密的私钥文件
